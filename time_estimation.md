## STEP 5: TIME ESTIMATION

**Assumption**:  
- 1–2 students working part-time.  
- Around 2–3 hours per day.  
- Focus is on completing a working demo with basic features.

### 5.1 High-Level Duration

- **Total Development Time (End-to-End)**:  
  - Approximately **4–6 weeks** (part-time), or  
  - Approximately **80–120 hours** overall.

### 5.2 Breakdown by Phase

- **Planning and Analysis**:
  - Tasks:
    - Finalize problem statement, objectives, and scope.
    - Identify main features and data needed.
    - Prepare basic UI sketches and architecture diagrams.
  - Estimated Time:
    - **8–12 hours** (1–1.5 weeks part-time).

- **Backend Development (Django REST Framework)**:
  - Tasks:
    - Set up Django project and apps.
    - Design and implement models (Pharmacy, Medicine, Stock, Reservation, etc.).
    - Seed demo data for Goa pharmacies.
    - Implement APIs:
      - Location suggestion.
      - Medicine search.
      - Nearest pharmacy calculation.
      - Pharmacy CRUD.
      - Reservation handling.
    - Implement conceptual OCR mock endpoint (if included).
  - Estimated Time:
    - **30–40 hours**.

- **Frontend Development (React)**:
  - Tasks:
    - Set up React project structure.
    - Implement pages/screens:
      - Home page with location input and suggestions.
      - Medicine search and results display.
      - Pharmacy details and reservation flow.
      - Simple pharmacy portal for CRUD operations.
    - Integrate with backend APIs.
    - Handle basic validation and error messages.
  - Estimated Time:
    - **30–40 hours**.

- **Testing and Debugging**:
  - Tasks:
    - Unit-level testing for APIs (basic).
    - Manual testing of user flows:
      - Enter location → search medicine → view pharmacies.
      - Upload prescription (if included) → confirm medicines → search.
      - Pharmacy login (if present) → manage profile and stock.
    - Fixing bugs, improving error messages.
  - Estimated Time:
    - **10–15 hours**.

- **Documentation and Final Preparation**:
  - Tasks:
    - Prepare project report (introduction, system analysis, design, implementation).
    - Prepare user manual (how to run and use the system).
    - Prepare viva-ready notes (key design decisions, assumptions, limitations).
    - Create basic diagrams (use case, architecture, data model).
  - Estimated Time:
    - **8–12 hours**.

### 5.3 Summary of Time Estimation

- **Planning**: 8–12 hours  
- **Backend Development**: 30–40 hours  
- **Frontend Development**: 30–40 hours  
- **Testing**: 10–15 hours  
- **Documentation**: 8–12 hours  

**Total** (approximate): **86–119 hours**, which fits into **4–6 weeks** of part-time work.
